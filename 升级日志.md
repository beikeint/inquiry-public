## v1.0 · 询盘全链路闭环（2026-04-14 创建）

**事实**：询盘管理智能体 v1.0 于 2026-04-14 创建，位于 `智能体/运营/询盘-inquiry/`，6 个核心技能，与网站运营-web-ops通过 client-manager 协作做转化归因闭环。

**Why**：网站运营-web-ops在"获客层"已经接近天花板，但**转化归因断裂** — 能追踪到"有人点了 WhatsApp"，但不知道最终是否成交。需要独立智能体管理询盘 → 成交全链路，并将成交数据回传给网站运营做内容归因。

### 6 个核心技能

| 技能 | 职责 |
|---|---|
| `lead-capture` | 表单/WhatsApp/邮件/社媒询盘统一入口 |
| `lead-enrichment` | 公司信息 / 采购决策人 / 采购意图评分 |
| `email-nurture` | 自动邮件培育序列（初次 → 跟进 → 报价 → 成交） |
| `outbound-prospect` | 主动获客：Apollo 线索 + 个性化开发信 |
| `deal-tracking` | 成交跟踪 + 归因数据生成 |
| `weekly-pipeline` | 周度管道报表：进度 / 瓶颈 / 风险 |

### 跨智能体协作架构

- 两个智能体通过 **client-manager 时间线** 协作，各管各的职责
- **网站运营不碰询盘跟进，询盘-inquiry不碰 SEO** — 职责严格分离
- **内容归因报告**是两者的数据桥：`deal-tracking` 技能生成，网站运营月报引用

### MCP 复用

client-manager（共享）· email · apollo · linkedin · fetch · memory

### How to apply

- 入口：`智能体/运营/询盘-inquiry/`（独立 git 仓库）
- 新客户接入：client-manager 里打标"需询盘管理" → 自动路由到此智能体
