# 周度管道报告

> 触发指令：`管道报告` / `[客户] 管道报告`  
> 执行频率：每周一  
> 人工确认：无需确认  
> 耗时：全局约10分钟 / 单客户约5分钟

---

## 执行流程

### Step 1: 拉取所有线索数据

```
并行执行：
├→ client-manager.list_clients(active)
└→ 对每个客户：
   └→ client-manager.get_client(id) → 过滤时间线中所有"询盘"相关记录
```

### Step 2: 分阶段统计

```
遍历所有线索，按当前阶段分类：

pipeline = {
  "New Lead": [],        // 新进来还没响应的
  "Contacted": [],       // 已响应，等对方回复
  "Qualified": [],       // 已确认需求
  "Quoted": [],          // 已报价，等结果
  "Negotiating": [],     // 在谈判
  "Won": [],             // 本周成交的
  "Lost": [],            // 本周流失的
  "On Hold": []          // 暂缓的
}
```

### Step 3: 预测本月营收

```
预测方法（加权管道法）：
├─ Qualified 阶段的线索 × 30% 概率 × 预估金额
├─ Quoted 阶段的线索 × 50% 概率 × 报价金额
├─ Negotiating 阶段的线索 × 70% 概率 × 谈判金额
└─ 总和 = 本月预测营收

例：
  3条 Qualified × 30% × $40,000 = $36,000
  2条 Quoted × 50% × $50,000 = $50,000
  1条 Negotiating × 70% × $45,000 = $31,500
  预测：$117,500
```

### Step 4: 生成加速建议

```
扫描所有活跃线索，找出需要行动的：
├─ 🔴 Quoted 超过7天无进展 → "催一下报价反馈"
├─ 🔴 Qualified 超过3天未报价 → "赶紧出报价"
├─ 🟡 Contacted 超过48小时无回复 → "换一种方式跟进"
├─ 🟡 On Hold 超过30天 → "重新激活试试"
└─ 🟢 本周新进的Hot线索 → "优先跟进"
```

## 注意事项

- **数据来源是 client-manager 时间线**：所有线索状态必须及时更新到时间线，否则报告不准
- **预测是概率加权，不是承诺**：明确标注"预测"，不能让客户以为是确定营收
- **每周一早上出**：和网站运营的周检同一天，运营人员一次看完两个报告
- **加速建议必须可执行**：不能说"加快跟进"，要说"给Mehmet发WhatsApp问报价反馈"
- **趋势比绝对数重要**：连续3周下降比单周数字低更值得关注
